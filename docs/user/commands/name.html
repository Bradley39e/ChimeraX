Command: name

Usage:
name  target-name  spec

Usage:
name  frozen  target-name  spec

Usage:
name  list

Usage:
name  delete ( target-name | all )

The name command defines a target-name to be used as a synonym for spec in subsequent commands. This can be used to replace a long target specification string with something shorter and easier to type, or to name the current selection for easy access to its contents later, regardless of what is selected at that time. Subsequent occurrences of target-name (space- and/or semicolon-delimited) in commands are translated to the corresponding items. See also: alias, rename

A target can be defined as evaluate-now or evaluate-later:

  • An evaluate-later target created with name is simply expanded to the original spec string when it is used.
  • An evaluate-now target created with name frozen is defined as only the currently evaluated contents of spec, which could be sel to indicate the current selection. For example, if a selection of all oxygens in a specific atomic model is defined as a target with:
    select #2&O
    name frozen proto sel
    ...the target proto will not be affected by any subsequent changes in what is selected, nor will it include the oxygens of any other atomic model, not even the same structure if it was closed and reopened as the same model number.

The command name list lists the names of all user-defined targets in the Log. The expansion of any evaluate-later target can be shown with name target-name.

The command name delete can be used to “forget” a specific user-defined target or all user-defined targets.
