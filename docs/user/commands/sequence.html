Command: sequence

Usage:
sequence  chain  chain-spec

Usage:
sequence  associate  chain-spec [alignment:target]

Usage:
sequence  dissociate  chain-spec  alignment

The command sequence chain shows the sequence of the specified chain in the Sequence Viewer. Only one structure chain should be specified. See also: open, Molecule Display icon

Sequence-structure association (such as for synchronized selection) occurs automatically, but the commands sequence associate and sequence dissociate allow more precise control, for example, of which structure chains are used for header calculations, or forcing or removing associations regardless of whether the number of mismatches would be tolerated by the automatic procedure.

The command sequence associate associates one or more structure chains with a sequence. The target sequence for association is specified by alignment name, as shown in title bar of the Sequence Viewer window, and the name or index number of the target sequence in the alignment, in the form:  alignment:target  (enclosed in quotation marks if alignment or target includes spaces). The index number can be a positive integer N for the Nth sequence in the alignment, or negative for the Nth-to-last sequence.

Alternatively, the target can be omitted to associate each specified structure chain with the the best-matching sequence in the alignment. The alignment can be omitted if only one alignment is open, or if the target is also omitted; in the latter case, each specified structure chain will be associated with the best-matching sequence in each open alignment. If either or both are omitted, the colon (:) is not needed except in rare cases to disambiguate an alignment and sequence with the same name.

For sequence dissociate, only the alignment needs to be specified, not an individual sequence, because a structure chain can only be associated with one sequence per alignment.
