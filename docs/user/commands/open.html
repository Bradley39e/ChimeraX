Command: open

Usage:
open  filename [ format  file-format ] [ name  model-name ]

Usage:
open [ prefix:]identifier  [ fromDatabase  source ] [ format  file-format ] [ ignoreCache true | false ] [ name  model-name ]

Usage:
open formats

The command open reads local files and fetches data from the Web. Input files may contain 3D data to be displayed, or commands or code to be executed. See also: close, save, sym, list, initial colors

Any resulting model in ChimeraX is automatically assigned a model ID number, starting with #1. The name of the model (shown in the Model Panel) will be generated from the filename or database identifier unless another name is given with the name option. If model-name contains spaces, it should be enclosed in quotation marks.

The open formats command lists the possible formats and databases in the Log.

Examples:

open ~/Desktop/maps/myfile.mrc name refined3
open 2gbp
open 1zik format pdb
open emdb:3087
open 3237 from emdb name "gamma-secretase complex"

←Local Files

←Web Fetch

Alternatively, an entry can be fetched from a database if its identifier is supplied. The source database can be indicated with either a prefix before the identifier or the fromDatabase option. The default database for a 4-character identifier without prefix is the RCSB Protein Data Bank; the default format for that database is mmcif, although pdb can be specified with the format option. For other databases, there is no choice of format and no reason to use the format option.

database prefix source data type file-format
RCSB Protein Data Bank pdb: pdb atomic coordinates and associated annotations mmcif (default) or pdb
Electron Density Server eds: eds (2Fo-Fc) electron density map for a PDB entry;
not available for all PDB entries		 dsn6
Electron Microscopy Data Bank emdb: emdb density map ccp4
cellPACK cellpack: cellpack hierarchical model;
see cellPACK Chimera instructions for available identifiers		 cellpack

Web-fetched data are automatically cached locally in subdirectories of ~/Downloads/Chimera. Retrieving previously fetched data from the cache can be bypassed with ignoreCache true, although the new data will still be cached.
