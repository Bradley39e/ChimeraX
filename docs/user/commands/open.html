Command: open

Usage:
open  filename

The command open reads local files and fetches data from the Web. Input files may contain 3D data to be displayed, or commands or code to be executed. Any resulting model in ChimeraX is automatically assigned a model ID number, starting with #1 (no model #0). See also: list, close

Examples:

open ~/Desktop/maps/myfile.mrc
open 2gbp
open emdb:3087

←Local Files

The filename can be the pathname (including directory location) of a local file, with format indicated by filename suffix. Known formats include:

←Web Fetch

Alternatively, the filename can be the database identifier (ID) of an entry to fetch, with the database indicated by a prefix before the ID. The default for a 4-character ID without prefix is to fetch mmCIF from the Protein Data Bank.

database prefix data type
Protein Data Bank (mmCIF format) mmcif: atomic coordinates and associated annotations
Protein Data Bank (PDB format) pdb: atomic coordinates and associated annotations
Electron Density Server (2fo-fc) eds: crystallographic density map for a PDB entry;
not available for all PDB entries
Electron Microscopy Data Bank emdb: density map

Web-fetched data are automatically cached locally in subdirectories of ~/Downloads/Chimera.
