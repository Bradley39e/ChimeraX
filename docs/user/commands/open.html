Command: open

Usage:
open  filename [ format  format-name ] [ name  model-name ] [ smartInitialDisplay true | false ] [ trajectory true | false ]

Usage:
open [ prefix:]identifier  [ fromDatabase  source ] [ format  format-name ] [ name  model-name ] [ smartInitialDisplay true | false ] [ trajectory true | false ] [ ignoreCache ]

Usage:
open formats

The command open reads local files and fetches data from the Web. Input files may contain 3D data to be displayed, or commands or code to be executed. Alternatively, local files can be browsed/opened with File... Open in the menu, in which case the proper filename suffix is required. See also: rename, close, save, help, sym, initial colors

Any resulting model(s) in ChimeraX are automatically assigned a model ID number, starting with #1. The name of the model (shown in the Model Panel) will be generated from the filename or database identifier unless another name is given with the name option. If model-name contains spaces, it should be enclosed in quotation marks.

Setting smartInitialDisplay to false bypasses chemical perception and rule-based changes in lighting, coloring, and representation, instead always showing atomic models as white spheres; it may be useful in rare cases where generating the initial representation is slow (e.g., drawing cartoons for very large numbers of atomic models).

A multi-model PDB file can be opened as a trajectory (for sequential playback with coordset) by setting trajectory to true.

The open formats command lists the possible formats and databases in the Log.

Examples:

open ~/Desktop/maps/myfile.mrc name refined3
open 2gbp
open 1zik format pdb
open emdb:3087
open 3237 from emdb name "gamma-secretase complex"

Local Files

Usage: open  filename [ format  format-name ] [ name  model-name ] [ smartInitialDisplay true | false ] [ trajectory true | false ]

The filename is generally a pathname to a local file, either absolute or relative to the current working directory as reported by pwd. The working directory can be changed with cd. A pathname may start with “~” to indicate the user's home directory. One or more files can be specified at a time by including * (wild card), ? (single-character wild card), and/or square-bracketed character ranges in the filename (Unix-style pathname expansion or globbing).

The file format can be indicated with either the filename suffix or the format option (where format-name can be truncated). Files that are gzipped, as indicated by their additional .gz suffix, can also be read.

Web Fetch

Usage: open [ prefix:]identifier  [ fromDatabase  source ] [ format  format-name ] [ name  model-name ] [ smartInitialDisplay true | false ] [ trajectory true | false ] [ ignoreCache ]

An entry can be fetched from a database if its identifier is supplied. The source database can be indicated with either a prefix before the identifier or the fromDatabase option. The default database for a 4-character identifier without prefix is the RCSB Protein Data Bank; the default format for that database is mmcif, although pdb or mmtf can be specified with the format option. For other databases, there is no choice of format and no reason to use the format option.

database prefix source description format-name

see cellPACK Chimera instructions for available identifiers		 cellpack

Web-fetched data are stored locally in the standard user cache directory (platform-dependent, for example ~/Library/Caches/ChimeraX on a Mac; see python appdirs). The default download location for Chimera (for example, ~/Downloads/Chimera on a Mac) is checked first, and then this standard ChimeraX location. If the data file is not found locally or the user has specified ignoreCache, the file will be fetched and cached.
