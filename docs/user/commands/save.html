Command: save

Usage:
save [ image ]  filename [ format  file-format ] [ supersample N ] [ quality M ] [ width  w ] [ height  h ] [ transparentBackground true | false ]

Usage:
save [ session ]  filename [ format  session ]

The command save saves an image file or a ChimeraX session file to filename, which can be a pathname (including directory location). The pathname can be absolute or relative to the current working directory as reported by the command pwd. See also: open, log, movie, Graphics icons

The image or session keyword before the filename is optional if the file type is specified with either the filename suffix or the format option:

file-format (image) filename suffix description
png
(default for save image)		 .png Portable Network Graphics
tiff .tif Tagged Image File Format
jpeg .jpg Joint Photographic Experts Group
gif .gif Graphics Interchange Format
bmp .bmp bitmap
ppm .ppm Portable PixMap
file-format (session) filename suffix description
session .cxs ChimeraX session

If the format is specified separately, the appropriate suffix will be added to filename. In the case of a session, the filename will end in .cxs.gz because the file will be gzipped automatically. ChimeraX can open a session file in either the gzipped or unzipped state.

Image Options

Images are rendered offscreen. Images can be supersampled, that is, initially generated at a higher resolution and then sampled down to the final size. The degree of supersampling N can be specified with the supersample keyword; the default is 3, meaning that the initial image will be 3 times larger in each dimension, with 9 times as many pixels, as the final result. A value of 1 corresponds to no supersampling.

The quality setting applies to JPEG format only, with values of M in the range 0–100 (default 95).

The width and height values can be given in pixels. If neither is specified, the image will have the same pixel dimensions as the graphics window. If only one dimension is specified, the other will be set according to the graphics window aspect ratio; if both are specified but the image aspect ratio is not the same as the graphics window aspect ratio, the image contents will differ from what is shown in the window.

The transparentBackground option (default false) indicates whether to include opacity values (alpha) in an output PNG or TIFF file. With background transparency, the background will be completely invisible in the saved images. This facilitates combining the image with different backgrounds in image-editing applications. (Note: TIFF images with background transparency may not be interpreted correctly by Adobe Photoshop.) Synonyms for true: True, 1. Synonyms for false: False, 0.
