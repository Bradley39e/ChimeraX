Command-Line Color Specification

Colors can be specified in commands (color, etc.) with:

  • CSS color names except in lowercase only, optionally with spaces between words
    • grouped by similarity
    • listed alphabetically
  • color names defined previously with the colordef command
  • hexadecimal format starting with # and encoding rgb (3 or 6 digits) or rgba (4 or 8 digits)
  • CSS-like specifications in the following formats:
    • rgb(red, green, blue)
    • rgba(red, green, blue, alpha)
    • hsl(hue, saturation, lightness)
    • hsla(hue, saturation, lightness, alpha)
    • gray(grayscale)
    The colorspace name must be in lowercase and immediately precede the open parenthesis. Otherwise, embedded spaces are allowed. Values for red, green, blue, alpha (increasing transparent → opaque), saturation, lightness, and grayscale (increasing black → white) can be given as:
    • percentages indicated with %
    • fractional values in the range 0-1, indicated with a decimal point (for example, 1. rather than 1)
    The red, green, blue, and grayscale components can also be expressed as whole numbers in the range 0-255. Values for hue are in the range 0-360, corresponding to angular positions on the color wheel.

Color names from CSS or colordef can be truncated, with ambiguous truncations going to the shortest of the matching full color names.

Examples

The following all specify the same color:

red
#f00
#FF0000FF
rgb(255, 0, 0)
rgb(1., 0., 0.)
rgb(100%, 0%, 0%)
rgba(100%, 0%, 0%, 1)
rgba(100%, 0%, 0%, 100%)
hsl(0, 1.0, 0.5)
hsl(0, 100%, 50%)
The following specify the same shade of gray:
gray(128)
gray(50%)
