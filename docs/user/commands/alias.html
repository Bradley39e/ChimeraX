Command: alias

Usage:
alias [ name [ wordlist ]]

Usage:
~alias [ name ]

The alias command defines a new command name as the specified wordlist, which could be a composite of other commands. That is, wordlist may contain multiple commands separated by semicolons. Subsequent uses of name as a command (at the beginning of a line, following a semicolon, or as a command argument of perframe) will be replaced with wordlist. For user-defined targets based on specification strings or the current selection, see the namespec command (not yet implemented). See also: perframe

The alias command without arguments lists the existing aliases in the Log, and the definition of a specific alias can be shown with alias name.

The command ~alias without arguments deletes all user-defined aliases, whereas a specific alias can be deleted with ~alias name.

Within wordlist, $1, $2, $3 ... may be used to indicate the first, second, third ... arguments of name. To have a string with spaces in it handled as a single argument, enclose the string in single or double quotation marks. The quotation marks will be omitted when the alias is expanded. For example,

alias ribcolor color $1 $2 targ c
ribcolor "/a & helix" "dodger blue"
is equivalent to
color /a & helix dodger blue targ c
