Command: view

Usage:
view  atom-spec [ orient ]

Usage:
view  name  view-name

Usage:
view  view-name [ frames ]

Usage:
view  delete ( view-name | all )

Usage:
view  list

The view command moves the camera so that the specified items fill the graphics window. It also allows naming and later restoring the camera viewpoint. See also: cofr, zoom

Only the displayed parts of the specified items are used for calculating the new view. If nothing is specified, all displayed items will be included. The center of rotation is fixed at the center of the bounding box of the specified items (displayed parts only), or if nothing is specified, allowed to vary according to the front center method. The orient option rotates the camera to a standard orientation with scene X-axis horizontal increasing rightward, Y-axis vertical increasing upward, and Z-axis perpendicular to the screen increasing toward the viewer.

The name option assigns view-name to the current camera viewpoint (position and orientation). Subsequently, that viewpoint can be restored over a specified number of image update frames (default 1 frame). The delete option can be used to “forget” a specific viewpoint or all viewpoints. The list option reports the names of all currently saved viewpoints in the Log, and clicking a name therein restores the corresponding viewpoint.

Transition between Camera Viewpoints

The camera is rotated with a linear change in angle from its orientation in the first view to its orientation in the second view. At the same time, the camera is translated so that the center of rotation from the first view moves in a straight line to the center of rotation in the second view.
