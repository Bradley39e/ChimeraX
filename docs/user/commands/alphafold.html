Command: alphafold

Usage:
alphafold fetch  uniprot-id  [ colorConfidence  true | false ] [ ignoreCache  true | false ]

Usage:
alphafold match  chain-spec  [ search  true | false ] [ trim  true | false ] [ colorConfidence  true | false ] [ ignoreCache  true | false ]

The alphafold command fetches theoretical protein structures predicted by the AlphaFold AI system from the AlphaFold Database. Users of this resource should cite:

Highly accurate protein structure prediction with AlphaFold. Jumper J, Evans R, Pritzel A, et al. Nature. 2021 Jul 15. doi: 10.1038/s41586-021-03819-2.

The AlphaFold Database does not cover all sequences in UniProt, so there may not be a prediction available for a given protein chain. It is also important to realize that the predicted structures vary in confidence levels and should be interpreted with caution. AlphaFold predicts structures for single chains, not complexes, so assembling the individual predictions into a complex will frequently give unphysical results where parts of the chains intersect or interact poorly with one another. See also: Modeller Comparative

Which structures to fetch can be specified in two main ways.

  1. by UniProt accession number using alphafold fetch or open, for example:
    alphafold fetch p29474
    – OR – (equivalent)
    open p29474 from alphafold
  2. by chain(s) of an experimentally determined atomic model already open in ChimeraX, for example:
    alphafold match #1
    alphafold match #3/B,D trim false

    Giving a model number specifies all of its protein chains. For each chain, alphafold match:

    1. retrieves the prediction, if any, from the AlphaFold Database for that chain's UniProt accession number identified according to the search setting
    2. makes the chain ID of the predicted structure the same as the corresponding chain of the existing model
    3. superimposes the predicted structure on the corresponding chain of the model using matchmaker defaults

    For each retrieved prediction, the following are reported in a table in the Log:

    • Chain – chain ID
    • UniProt Name and UniProt Id (accession number)
    • RMSD – Cα root-mean-square deviation between the predicted and experimental structures, over all residues of the latter
    • Length – number of residues in the predicted structure
    • Seen – number of residues with atomic coordinates in the experimental structure

Options

colorConfidence  true | false
Whether to color the predicted structures by the pLDDT confidence measure in the B-factor field (default true):
  • 90
    to 100
    – high accuracy expected
  • 70
    to 90
    – expected to be modeled well (a generally good backbone prediction)
  • 50
    to 70
    – low confidence, treat with caution
  • 0
    to 50
    – should not be interpreted, may be disordered
...in other words, using palette 0,red:50,orange:70,yellow:90,cornflowerblue:100,blue
ignoreCache  true | false
The fetched predictions are stored locally in ~/Downloads/ChimeraX/AlphaFold/, where ~ indicates a user's home directory. If a file specified for opening is not found in this local cache or ignoreCache is set to true, the file will be fetched and cached.
search  true | false
When fetching predictions with alphafold match, whether to BLAT search the 21 proteomes in the database for the closest match if the UniProt accession number for a chain is not provided in the experimental structure's input file, or is provided but not found in the AlphaFold Database (true, default). The closest sequence match for which a prediction is available will be retrieved, as long as the sequence identity is at least 25%. With search false, only the experimental structure's input file will be used as a potential source of UniProt accession numbers. When present, these are given in DBREF records in PDB format and in struct_ref and struct_ref_seq tables in mmCIF.
trim  true | false
When fetching predictions with alphafold match, whether to trim the predicted protein structures to the same residue ranges as used in the experiment (true, default). These ranges are given in DBREF records in PDB format and in struct_ref and struct_ref_seq tables in mmCIF. Use trim false to include the full UniProt sequences, which could be longer.
