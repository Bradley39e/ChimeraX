Command: log

Usage:
log [ show | hide ] [ clear ] [ thumbnail [ width w ][ height h ]] [ save  pathname ] [ errorDialog true | false ] [ warningDialog true | false ] [ text | html character string ]

The command log acts on the Log tool. Options:

  • show or hide - open or close the Log
  • clear - clear the Log
  • save - save contents to an HTML file of specified pathname
  • thumbnail - include a thumbnail image of the current display; pixel width and height can be specified (default 100 x 100)
  • errorDialog - whether errors should elicit both a separate dialog and a message in the Log (default true) instead of just a message
  • warningDialog - whether warnings should elicit both a separate dialog and a message in the Log (default false) instead of just a message
  • text - echo the rest of the command (up to the next carriage return or semicolon, including any spaces) in the Log; must be the last option in the command
  • html - render the rest of the command (up to the next carriage return or semicolon, including any spaces) as HTML in the Log; must be the last option in the command, for example:
    log thumb html <b>Toxin from <i>Bacillus thuringiensis</i></b>
