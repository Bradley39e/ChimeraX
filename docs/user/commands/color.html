Command: color

Usage:
color  atom-spec  colorname   general-options

Usage:
color  atom-spec  sequential ( residues | chains ) [ cmap  colormap ]  general-options

Usage:
rainbow  atom-spec ( residues | chains ) [ cmap  colormap ]  general-options

The color command colors atomic models, including cartoon and molecular surface representations. The absence of an atom or model specifier implies “all.” Bonds and pseudobonds are considered as specified when both flanking atoms are specified. See examples below. See also: transparency, volume, initial colors

Subcommands of color:

  • color name - define color names, show existing definitions in the Log
  • color list - list color names in the Log
  • color delete - forget user-defined color names

For simple coloring, the colorname can be:

  • a color specification by name, hexadecimal color code, or component values
  • byelement or byatom - a color-by-element scheme; only applies to atomic representations and per-atom patches of molecular surfaces
  • byhet or byhetero - same as byelement, except leaving carbon atoms unchanged
  • fromatoms - color other targets (molecular surface patches, etc.) to match the corresponding atoms
  • bychain - a stereotyped scheme for coloring by chain ID, also used for the initial colors of per-chain molecular surfaces
  • bymodel - the initial colors of atomic models
The sequential method applies a range of colors (a colormap) to items at the specified level:
  • residues - a different color for each residue in a biopolymer chain (always applies to entire chains)
  • chains - a different color for each biopolymer chain in an atomic model (always applies to entire models)
Only the biopolymer chains are colored, not ligand and solvent residues, even if they have the same chain ID as a biopolymer chain.

The rainbow command is a synonym for color sequential (see examples).

General Options

Keywords can be truncated to unique strings. Synonyms for true: True, 1. Synonyms for false: False, 0.

The transparency option sets transparency to the specified percent, where 0% is completely opaque and 100% is completely transparent. Existing transparency is not changed unless this option is used or the colorname explicitly specifies alpha (opacity). An alpha value overrides the transparency  percent if both are given.

The target option allows coloring only a subset of the possible displays of a given atom, residue, etc. The string can be any combination of the following letters, without commas or spaces:

  • a - atoms/bonds (coloring shown for bonds in halfbond mode)
  • b - bonds-only (coloring shown for bonds not in halfbond mode)
  • c - cartoons (divisible to per-residue segments)
  • s - surfaces (molecular surfaces divisible to per-atom patches)
  • p - pseudobonds-only (coloring shown for pseudobonds not in halfbond mode)
The default target is abcs, except that surface coloring is not yet implemented for sequential coloring (if atoms have been colored sequentially, however, the surface can be colored to match the atoms with the command: color fromatoms target s).

By default, bonds are shown in halfbond mode, with color and transparency settings from the flanking atoms. Turning halfbond mode off for a bond (or pseudobond) reveals its own color/transparency settings independent of the atoms. The halfbond option can be used to toggle halfbond mode regardless of whether a color is given in the command. This option affects only the specified bonds (not specified pseudobonds), except obeying the target when it includes p.

Colormap Options

The cmap option specifies a colormap for sequential coloring as one of the following:

  • the name of a pre-defined palette (colors listed in ascending value order):
    • rainbow (default) - blue, cyan, green, yellow, red
    • red-white-blue or redblue
    • blue-white-red or bluered
    • cyan-white-maroon or cyanmaroon
    • grayscale or gray - black, white

  • color1:color2:color3... a series of colon-separated color specifications, collectively enclosed in quotation marks if it contains any spaces (example: cmap cornflowerblue:red:gold or cmap "cornflower blue:red:gold"). The colors are placed evenly in ascending value order along the range, with the first and last colors at the extremes. Colors for intermediate values are interpolated.

  • value1,color1:value2,color2:value3,color3... a series of colon-separated value,color-spec pairs, collectively enclosed in quotation marks if it contains any spaces (example: cmap "0,cornflower blue:1,gold:0.7,red"). For sequential coloring, an entire range (such as the residues in each chain or the chains in each model) is mapped to fractional values 0.0-1.0. Values below the lowest given will map to the same color as the lowest, and values above the highest will map to the same color as the highest. Colors are interpolated between specified value-color pairs; discrete coloring can be achieved by specifying the same color for the upper- and lower-bound values of a “bin.”

  • colourlovers-id - the ID number of a palette to be fetched from COLOURlovers.com. Navigating to the page of a specific palette at COLOURlovers.com shows its ID (in the URL) as well as any licensing stipulations for its use. Palettes with “all rights reserved” cannot be fetched. In ChimeraX, the palette colors are used in left-to-right order, spaced equally along the range of values. Web-fetched data are automatically cached locally in subdirectories of ~/Downloads/Chimera.

Examples

Example color commands:

color /A,B:12,260-275  hot pink  target a
- color residues 12 and 260-275 in chains A and B, atoms/bonds only
color pbonds  yellow  targ p
- color all pseudobonds yellow, but not their flanking atoms
color  seq res
– or –
rainbow res
- rainbow-color residues blue → red along each biopolymer chain
color #3  seq chain  cmap cyan
– or –
rainbow #3  chain  cmap cyan
- color the chains in model 3 from a medium cyan through white to dark maroon
color  seq chain  targ c  cmap cornflowerblue:red:gold
– or –
rainbow  chain  targ c  cmap cornflowerblue:red:gold
- color the chains in each atomic model, cartoons only, along the range cornflower blue → red → gold

Defining and Listing Colors

Usage:
color name  cname [ color-spec ]

Usage:
color list [ all ]

Usage:
color delete ( cname | all )

The command color name defines a new color name or redefines an existing one. The name is assigned as the name of the specified color. Color names must begin with an alphabetical letter and may contain only lowercase letters, numbers, hyphens, underscores, and spaces. If name has embedded spaces, it must be enclosed in quotation marks.

Examples:
color name tpurple 60,0,80,50
color name tgreen rgba(0%, 80%, 40%, 0.5)

The command color list shows any user-defined colors in the Log, whereas color list all also lists the built-in color names. The rgba (red, green, blue, opacity) definition of an existing color can be shown in the Log with color name cname.

The delete option can be used to “forget” a specific user-defined color or all user-defined colors. A built-in color name cannot be deleted.
